# الأحداث في Solidity: فهم الأحداث في لغة البرمجة Solidity

## ملخص
الأحداث هي ميزات في لغة Solidity تسمح للعقود الذكية بإصدار إشعارات إلى التطبيقات اللامركزية عند حدوث تغييرات معينة في حالة العقد.

## الوثائق
### الغرض
تستخدم الأحداث في Solidity لتسهيل الاتصال بين العقد الذكية وواجهات المستخدم أو التطبيقات اللامركزية. عندما يتم استدعاء حدث معين، يمكن للتطبيقات الخارجية أن تلتقط هذا الحدث وتتعرف على تغييرات الحالة التي حدثت داخل العقد.

### الاستخدام
يتم تعريف الأحداث باستخدام الكلمة الرئيسية `event`، ويمكن استدعاؤها في أي مكان داخل العقد الذكي. عند إصدار حدث، يتم تسجيله في السجل، مما يجعل من الممكن استرجاعه لاحقًا.

#### صيغة تعريف الحدث:
```solidity
event EventName(Type1 indexed param1, Type2 indexed param2);
```

#### استدعاء الحدث:
```solidity
emit EventName(value1, value2);
```

### التفاصيل
- **الوسوم (Indexed Parameters)**: يمكنك استخدام الكلمة الرئيسية `indexed` لتسمية متغيرات معينة في الحدث، مما يسهل البحث عنها في السجلات.
- **التسجيل في السجل**: الأحداث يتم تسجيلها في "السجل" (logs)، والذي يختلف عن حالة العقد، مما يسمح بتقليل تكلفة التخزين.
- **الحد من البيانات**: يجب أن تكون الأحداث خفيفة، لذلك من الأفضل تجنب إضافة كميات كبيرة من البيانات إليها.

## الأمثلة
### مثال أساسي لتعريف واستخدام حدث
```solidity
pragma solidity ^0.8.0;

contract Example {
    event ValueChanged(uint256 newValue);

    uint256 public value;

    function setValue(uint256 newValue) public {
        value = newValue;
        emit ValueChanged(newValue); // استدعاء الحدث
    }
}
```

### مثال مع وسوم
```solidity
pragma solidity ^0.8.0;

contract ExampleWithIndexed {
    event DataSent(address indexed sender, string message);

    function sendData(string memory message) public {
        emit DataSent(msg.sender, message); // استدعاء الحدث
    }
}
```

## الشرح
عند استخدام الأحداث في Solidity، يجب مراعاة بعض النقاط:
- **التكلفة**: الأحداث أقل تكلفة من تخزين البيانات في حالة العقد، لذا يُفضل استخدامها لتقليل التكاليف.
- **الوصول**: لا يمكن استرجاع الأحداث من داخل العقود الذكية بعد أن يتم إصدارها، بل يجب على التطبيقات الخارجية التقاطها من السجل.
- **عدم إمكانية التعديل**: بمجرد إصدار حدث، لا يمكن تعديله أو حذفه، لذا يجب التأكد من دقة البيانات المرسلة.

## ملخص جملة واحدة
الأحداث في Solidity هي وسيلة فعالة للتواصل بين العقود الذكية والتطبيقات اللامركزية من خلال إصدار إشعارات عند حدوث تغييرات في حالة العقد.
