# العمومية في Solidity: فهم الكلمة المفتاحية "public"

## ملخص
الكلمة المفتاحية "public" في Solidity تُستخدم لتحديد الوصول إلى المتغيرات والدوال، مما يسمح بالوصول إليهما من أي مكان، سواء من داخل العقد أو من خارج العقد (مثل العقود الأخرى أو الواجهات الأمامية).

## الوثائق
تُعتبر "public" أحد محددات الوصول الأساسية في لغة Solidity. عند تطبيقها على متغير أو دالة، تعني أن هذا العنصر قابل للوصول من أي مكان. 

### الغرض
الغرض من استخدام "public" هو تسهيل التفاعل مع المتغيرات والدوال، مما يزيد من المرونة وسهولة الاستخدام.

### الاستخدام
يمكن استخدام "public" عند تعريف المتغيرات والدوال. عند تعريف متغير كـ "public"، يتم إنشاء دالة قراءة تلقائيًا، مما يسمح باسترجاع قيمته بدون الحاجة إلى كتابة دالة خاصة.

### التفاصيل
- **المتغيرات العامة**: تُستخدم "public" لجعل المتغيرات قابلة للوصول من خارج العقد. عند تعريف متغير كـ "public"، يتم إنشاء دالة تلقائية تعيد قيمة المتغير.
- **الدوال العامة**: تجعل "public" الدوال متاحة للاستخدام من تفاعلات العقود الأخرى أو المستخدمين من خلال الواجهة الأمامية.

## أمثلة
### مثال على متغير عام
```solidity
pragma solidity ^0.8.0;

contract MyContract {
    uint public myNumber; // متغير عام

    function setNumber(uint _number) public {
        myNumber = _number;
    }
}
```

### مثال على دالة عامة
```solidity
pragma solidity ^0.8.0;

contract MyContract {
    function greet() public pure returns (string memory) {
        return "Hello, World!"; // دالة عامة
    }
}
```

## الشرح
على الرغم من أن استخدام "public" يجعل المتغيرات والدوال سهلة الوصول، إلا أن هناك بعض النقاط التي يجب الانتباه إليها:

- **الأمان**: يجب استخدام "public" بحذر، حيث يمكن أن تعرض البيانات الحساسة للخطر إذا لم يتم التحكم في الوصول إليها بشكل صحيح.
- **التحكم في الوصول**: إذا كنت ترغب في تقييد الوصول إلى دالة، يمكنك استخدام "internal" أو "private" بدلاً من "public".

## ملخص جملة واحدة
تتيح الكلمة المفتاحية "public" في Solidity الوصول المفتوح إلى المتغيرات والدوال، مما يسهل التفاعل مع عقود Ethereum.
