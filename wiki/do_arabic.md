# استخدام الأمر "do" في لغة Solidity: دليل شامل

## ملخص
الأمر "do" في Solidity لا يعتبر جزءًا من الصياغة الأساسية للغة، بل هو جزء من تعبيرات التحكم مثل "do while"، والذي يُستخدم للتحكم في تدفق البرنامج. يعد هذا الأمر ذا أهمية خاصة عند الحاجة إلى تنفيذ كود معين على الأقل مرة واحدة.

## الوثائق
يستخدم "do" في عبارة "do while" لتكرار تنفيذ كتلة من التعليمات البرمجية طالما كانت الشرط المحدد صحيحًا. يتم تنفيذ الكود داخل الكتلة مرة واحدة على الأقل قبل تقييم الشرط.

### الاستخدام
تتكون عبارة "do while" من قسمين رئيسيين:
1. **كتلة التنفيذ**: الكود الذي سيتم تنفيذه.
2. **الشرط**: شرط يتم تقييمه بعد تنفيذ الكود.

الصياغة العامة هي:
```solidity
do {
    // كتلة من التعليمات البرمجية
} while (condition);
```

### التفاصيل
- **تنفيذ الكود**: يتم تنفيذ الكود داخل الكتلة على الأقل مرة واحدة بغض النظر عن الشرط.
- **تقييم الشرط**: بعد تنفيذ الكود، يتم تقييم الشرط. إذا كان صحيحًا، سيتم تنفيذ الكود مرة أخرى.
- **الإنهاء**: يستمر هذا التكرار حتى يصبح الشرط غير صحيح.

## أمثلة
### مثال 1: استخدام "do while" لطباعة الأرقام
```solidity
pragma solidity ^0.8.0;

contract NumberPrinter {
    function printNumbers() public pure returns (uint) {
        uint i = 0;
        do {
            i++;
        } while (i < 5);
        return i; // ستعيد القيمة 5
    }
}
```

### مثال 2: استخدام "do while" مع شرط معقد
```solidity
pragma solidity ^0.8.0;

contract Counter {
    function countUntil(uint limit) public pure returns (uint) {
        uint count = 0;
        do {
            count++;
        } while (count < limit);
        return count; // ستعيد القيمة المحددة
    }
}
```

## الشرح
### الأخطاء الشائعة
1. **عدم وجود شرط نهائي**: يجب التأكد من وجود شرط سيؤدي إلى إنهاء الحلقة. إذا كان الشرط دائمًا صحيحًا، ستستمر الحلقة إلى الأبد.
2. **عدم تعديل المتغيرات المستخدمة في الشرط**: تأكد من تعديل المتغيرات في الكتلة، وإلا ستبقى في حلقة لا نهائية.
3. **استخدام "do" خارج السياق المناسب**: يجب استخدام "do" فقط في تعبيرات التحكم المناسبة مثل "do while".

## ملخص في جملة واحدة
الأمر "do" في Solidity يُستخدم في تعبيرات التحكم لتكرار تنفيذ الكود على الأقل مرة واحدة قبل تقييم الشرط.
