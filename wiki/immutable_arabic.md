# الكلمة المفتاحية "immutable" في لغة Solidity: الاستخدامات والتفاصيل

## ملخص
تُعد الكلمة المفتاحية "immutable" في لغة Solidity ميزة جديدة تُستخدم لتعريف المتغيرات التي لا يمكن تعديلها بعد تعيين قيمتها في وقت الإنشاء. هذه الميزة تعزز الأمان والكفاءة في عقود Ethereum الذكية.

## الوثائق
### الغرض
تُستخدم الكلمة المفتاحية "immutable" لتحديد المتغيرات التي يمكن تعيين قيمتها فقط عند إنشائها، مما يجعلها مثالية للقيم الثابتة التي لا تتغير خلال حياة العقد.

### الاستخدام
يمكن استخدام "immutable" لتعريف المتغيرات في العقود الذكية. على عكس المتغيرات العادية، لا يمكن تعديل المتغيرات المعرفة بـ "immutable" بعد تعيين قيمتها. تُعتبر هذه الميزة فعالة من حيث الكفاءة حيث تقلل من تكلفة التخزين في الشبكة.

### التفاصيل
- تُستخدم الكلمة المفتاحية "immutable" مع المتغيرات فقط في مستوى العقد، ولا يمكن استخدامها مع المتغيرات المحلية أو مع المتغيرات في الوظائف.
- يجب تعيين قيمة المتغير "immutable" في المُنشئ (constructor) للعقد.
- يمكن استخدام "immutable" مع أنواع البيانات المختلفة مثل `uint`, `address`, و `string`.

## الأمثلة
### مثال بسيط
```solidity
pragma solidity ^0.8.0;

contract Example {
    // تعريف متغير immutable
    address public immutable owner;

    // المُنشئ لتعيين قيمة المتغير
    constructor() {
        owner = msg.sender; // تعيين قيمة المتغير عند إنشاء العقد
    }
}
```

### مثال آخر
```solidity
pragma solidity ^0.8.0;

contract Token {
    string public immutable tokenName;
    uint256 public immutable tokenSupply;

    constructor(string memory name, uint256 supply) {
        tokenName = name; // تعيين اسم الرمز
        tokenSupply = supply; // تعيين كمية الرمز
    }
}
```

## الشرح
### الفخاخ الشائعة
- **عدم القدرة على التعديل**: يُفترض أن يكون المطورون مدركين أن المتغيرات المعرفة بـ "immutable" لا يمكن تعديلها بعد تعيينها، مما قد يؤدي إلى أخطاء إذا تم استخدامها بشكل غير صحيح.
- **تعيين القيمة في المُنشئ فقط**: يجب أن تكون قيمة المتغير "immutable" متاحة أثناء تنفيذ المُنشئ، لذلك يجب التأكد من تعيينها بطريقة صحيحة.

### ملاحظات إضافية
- استخدام "immutable" يمكن أن يحسن من الأداء ويقلل من تكاليف التخزين.
- يُفضل استخدام "immutable" عند الحاجة إلى قيم ثابتة لا تتغير، مما يسهل الحفاظ على أمان العقد.

## ملخص جملة واحدة
تُستخدم الكلمة المفتاحية "immutable" في Solidity لتعريف المتغيرات التي لا يمكن تعديلها بعد تعيين قيمتها في المُنشئ، مما يعزز الأمان والكفاءة في العقود الذكية.
