# التعداد (enum) في لغة Solidity: تعريف واستخدامات

## ملخص
التعداد (enum) في Solidity هو نوع بيانات يُستخدم لتعريف مجموعة من القيم الثابتة، مما يسهل على المطورين تنظيم البيانات وتسهيل قراءتها.

## الوثائق
التعداد (enum) هو نوع بيانات خاص في Solidity يُستخدم لتعريف مجموعة من القيم الثابتة. يمكن استخدامه لتمثيل حالات مختلفة في العقود الذكية بطريقة أكثر وضوحًا وقراءة.

### الغرض
يُستخدم التعداد لتبسيط إدارة القيم الثابتة، مثل الحالات أو الخيارات المحددة. يمكن أن يساعد في جعل الكود أكثر وضوحًا وسهولة في الصيانة.

### الاستخدام
لتعريف تعداد، يتم استخدام الكلمة المفتاحية `enum` متبوعةً باسم التعداد وقائمة القيم. على سبيل المثال:

```solidity
enum Status { Pending, Active, Completed }
```

بعد تعريف التعداد، يمكن استخدامه كنوع بيانات في المتغيرات أو كجزء من هيكل بيانات أو وظيفة.

### التفاصيل
- **التعدادات تبدأ من 0**: كل قيمة في التعداد تُعطى قيمة رقمية تبدأ من 0. على سبيل المثال، في التعداد السابق، `Pending` تكون 0، `Active` 1، و`Completed` 2.
- **استخدام التعدادات**: يمكن استخدام التعدادات في الشروط والتكرارات، مما يجعل من السهل التعامل مع حالات متعددة.

## الأمثلة
### تعريف واستخدام تعداد
```solidity
pragma solidity ^0.8.0;

contract TaskManager {
    enum Status { Pending, Active, Completed }
    
    struct Task {
        string name;
        Status status;
    }
    
    Task[] public tasks;
    
    function createTask(string memory name) public {
        tasks.push(Task(name, Status.Pending));
    }
    
    function completeTask(uint index) public {
        tasks[index].status = Status.Completed;
    }
}
```

### الوصول إلى قيم التعداد
يمكن الوصول إلى قيم التعداد باستخدام الأرقام المقابلة أو الأسماء مباشرة:
```solidity
Status taskStatus = Status.Active; // استخدام الاسم
uint statusIndex = uint(Status.Completed); // استخدام الرقم
```

## الشرح
### الأخطاء الشائعة
- **تغيير القيم**: لا يمكن تغيير قيم التعداد بعد تعريفها. إذا كنت بحاجة إلى إضافة قيم جديدة، يجب عليك تعديل الكود وإعادة نشر العقد.
- **عدم استخدام التعدادات في الحلقات**: قد يؤدي استخدام التعدادات بشكل غير صحيح في الحلقات إلى أخطاء أو نتائج غير متوقعة.

### ملاحظات إضافية
- من المهم أن تكون القيم التي تُستخدم في التعداد واضحة ومفهومة، مما يسهل على المطورين الآخرين فهم الكود.
- يُفضل استخدام التعدادات بدلاً من الأعداد السحرية، حيث إن ذلك يحسن من قابلية قراءة الكود.
