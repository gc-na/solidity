# Solidity에서의 "pure" 함수: 정의와 사용법

## 개요
Solidity에서 "pure" 키워드는 함수의 특성을 정의하는 데 사용됩니다. 이 키워드는 함수가 상태 변수나 블록체인 상태에 접근하지 않고, 오직 입력값만을 사용하여 결과를 반환함을 나타냅니다. 

## 문서화

### 목적
"pure" 함수는 스마트 계약 내에서 계산을 수행할 수 있도록 설계되었습니다. 이러한 함수는 상태를 변경하지 않으며, 블록체인 상의 데이터에 접근하지 않기 때문에 효율적이고 안전한 코드 작성을 돕습니다.

### 사용법
Solidity에서 "pure" 키워드는 함수 선언에 사용됩니다. 함수 앞에 "pure" 키워드를 추가하여 해당 함수가 상태를 변경하지 않음을 명시합니다.

```solidity
function exampleFunction(uint256 input) public pure returns (uint256) {
    return input * 2;
}
```

### 세부 사항
- "pure" 함수는 입력 매개변수로 주어진 값만을 사용하여 결과를 반환합니다.
- 상태 변수나 블록체인 정보를 읽지 않기 때문에, 이러한 함수는 가스 비용이 적게 듭니다.
- "pure" 함수는 상태를 변경하지 않으므로, 다른 스마트 계약이나 외부 데이터에 의존하지 않습니다.

## 예제

### 기본 사용 예제
다음은 "pure" 함수를 사용하는 간단한 예제입니다.

```solidity
pragma solidity ^0.8.0;

contract Math {
    function multiply(uint256 a, uint256 b) public pure returns (uint256) {
        return a * b;
    }
}
```

위의 예제에서 `multiply` 함수는 두 개의 숫자를 곱하여 결과를 반환합니다. 이 함수는 상태를 변경하지 않고 입력값만을 사용합니다.

## 설명
"pure" 함수를 사용할 때 주의해야 할 점은 다음과 같습니다:
- 상태 변수에 접근하거나 수정하려고 하면 컴파일 오류가 발생합니다.
- "pure" 함수는 외부 계약의 상태에 접근할 수 없으므로, 데이터 전송이 필요한 경우에는 "view" 함수를 사용해야 합니다.
- "pure" 함수는 테스트와 디버깅을 용이하게 하며, 코드의 가독성을 높입니다.
