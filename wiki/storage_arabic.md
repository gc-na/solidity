# التخزين في Solidity: فهم كيفية إدارة البيانات على البلوكتشين

## ملخص
التخزين في Solidity هو أحد المفاهيم الأساسية التي تحدد كيفية تخزين وإدارة البيانات في العقود الذكية. يوفر Solidity عدة أنواع من التخزين، بما في ذلك التخزين الدائم (storage)، والمتغيرات المحلية (memory)، والمتغيرات المؤقتة (stack).

## الوثائق
تعتبر إدارة التخزين في Solidity من الجوانب الحاسمة عند تطوير العقود الذكية. يُستخدم التخزين بشكل أساسي لتخزين البيانات التي يحتاجها العقد على المدى الطويل، حيث يبقى هذا البيانات موجودة حتى بعد تنفيذ المعاملات. يتم تخزين البيانات في حالة التخزين الدائم على البلوكتشين، مما يعني أن أي تغيير يحدث سيبقى مسجلاً.

### أنواع التخزين:
1. **Storage**: 
   - يُستخدم لتخزين البيانات بشكل دائم. المتغيرات المُعلنة باستخدام `storage` يتم تخزينها في حالة البلوكتشين ويمكن الوصول إليها من خلال المعاملات المستقبلية.
  
2. **Memory**: 
   - يُستخدم لتخزين البيانات بشكل مؤقت. يتم مسح المتغيرات المُعلنة باستخدام `memory` عند انتهاء تنفيذ العقد.

3. **Stack**: 
   - يستخدم لتخزين البيانات بشكل مؤقت أثناء تنفيذ العمليات. يتميز بسرعة الوصول ولكن لديه حدود في الحجم.

### كيفية الاستخدام:
- لتحديد نوع التخزين، يمكنك استخدام كلمات مفتاحية مثل `storage` و `memory`. إليك بعض الأمثلة لتوضيح ذلك.

## أمثلة
### مثال 1: تخزين البيانات باستخدام storage
```solidity
pragma solidity ^0.8.0;

contract Example {
    uint256 public storedData; // هذا متغير من نوع storage

    function set(uint256 x) public {
        storedData = x; // تخزين القيمة في التخزين الدائم
    }
}
```

### مثال 2: استخدام memory
```solidity
pragma solidity ^0.8.0;

contract Example {
    function calculate(uint256 x) public pure returns (uint256) {
        uint256 tempData = x * 2; // تخزين مؤقت في الذاكرة
        return tempData;
    }
}
```

## الشرح
عند التعامل مع التخزين في Solidity، هناك بعض النقاط التي يجب مراعاتها:

- **تكاليف التخزين**: تخزين البيانات في البلوكتشين مكلف من ناحية الغاز. لذلك، يجب استخدام التخزين بحذر.
- **الوصول إلى البيانات**: عند استرجاع البيانات من `storage`، قد يتطلب الأمر وقتًا أكثر مقارنةً مع `memory`.
- **تحديث البيانات**: أي تحديث لبيانات في `storage` يتطلب تكلفة غاز إضافية، لذا يجب إجراء التحديثات بحذر.

## ملخص جملة واحدة
التخزين في Solidity هو الطريقة التي يتم بها إدارة البيانات بشكل دائم على البلوكتشين، مما يؤثر على التكلفة والأداء في العقود الذكية.
