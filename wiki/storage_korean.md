# Solidity 저장소(Storage): 스마트 계약의 데이터 관리 기초

## 개요
Solidity에서 저장소(Storage)는 스마트 계약이 데이터를 영구적으로 저장하는 공간을 의미합니다. 저장소는 블록체인에 영구적으로 기록되며, 이 데이터는 계약의 상태를 유지하는 데 필수적입니다.

## 문서화
### 목적
Solidity의 저장소는 스마트 계약의 상태 변수를 저장하는 데 사용됩니다. 이 저장소는 블록체인에 영구적으로 존재하며, 계약의 상태를 변경할 때마다 업데이트됩니다.

### 사용법
Solidity에서 저장소에 데이터를 저장하려면 `storage` 키워드를 사용하여 상태 변수를 정의합니다. `storage`는 계약의 상태를 나타내며, 이 데이터는 블록체인에 직접 저장됩니다.

```solidity
pragma solidity ^0.8.0;

contract Example {
    uint256 public count; // 상태 변수

    function increment() public {
        count += 1; // 상태 변수 업데이트
    }
}
```

### 세부 사항
- **초기화**: 상태 변수는 계약이 배포될 때 초기화됩니다. 초기값을 설정하지 않으면 기본값(예: 정수형은 0)이 할당됩니다.
- **가시성**: 상태 변수는 `public`, `private`, `internal`, `external` 키워드를 통해 접근 권한을 설정할 수 있습니다.
- **비용**: 저장소에 데이터를 추가하면 가스 비용이 발생합니다. 이는 블록체인에 데이터를 영구적으로 저장하는 데 드는 비용입니다.

## 예제
다음은 Solidity의 저장소를 사용하는 기본적인 예제입니다.

```solidity
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 public data; // 저장소에 저장되는 상태 변수

    function setData(uint256 _data) public {
        data = _data; // 저장소에 데이터 저장
    }

    function getData() public view returns (uint256) {
        return data; // 저장된 데이터 조회
    }
}
```

## 설명
- **가스 비용**: 저장소에 데이터를 저장하는 것은 가스 비용이 많이 들 수 있으므로, 불필요한 데이터 저장을 피하는 것이 좋습니다.
- **배포 시 초기화**: 계약이 배포될 때 상태 변수를 초기화하지 않으면 기본값이 사용되므로, 계약의 초기 상태를 신중히 설정해야 합니다.
- **데이터 타입**: 상태 변수는 다양한 데이터 타입을 사용할 수 있으며, 각 데이터 타입에 따라 저장소에서의 처리 방식이 달라질 수 있습니다.

## 요약
Solidity의 저장소는 스마트 계약의 상태를 영구적으로 관리하며, 상태 변수를 통해 블록체인에 데이터를 저장하고 업데이트하는 데 필수적입니다.
