# خاصية "private" في Solidity: التحكم في الوصول للبيانات

## الملخص
تُستخدم خاصية "private" في Solidity لتقييد الوصول إلى المتغيرات والدوال داخل العقود الذكية، مما يضمن أن هذه العناصر غير متاحة للاستخدام من عقود أخرى أو من خارج العقد.

## التوثيق
تعتبر خاصية "private" واحدة من أهم ميزات التحكم في الوصول في لغة Solidity، حيث تُحدد مستوى الوصول للمتغيرات والدوال داخل العقد الذكي. عند تعريف متغير أو دالة كـ "private"، فإنها تكون مرئية فقط داخل نفس العقد، مما يزيد من الأمان ويقلل من خطر الوصول غير المصرح به.

### الاستخدام
- **تعريف المتغيرات**: يمكن الإعلان عن المتغيرات كـ "private" لضمان عدم إمكانية الوصول إليها من عقود أخرى.
- **تعريف الدوال**: تُستخدم أيضًا لتحديد دوال لا يمكن استدعاؤها إلا من داخل العقد نفسه.

### تفاصيل
- عند استخدام "private"، يجب أن تكون على علم بأن المتغيرات والدوال المعرفة بهذه الصفة لن تكون قابلة للوصول من قبل أي عقد آخر، بما في ذلك العقود الوراثية.
- يُفضل استخدام "private" عندما لا يكون هناك حاجة للوصول إلى المتغيرات أو الدوال من خارج العقد، مما يُعزز الأمان.

## الأمثلة
### مثال 1: تعريف متغير خاص
```solidity
pragma solidity ^0.8.0;

contract MyContract {
    uint256 private secretNumber;

    function setSecretNumber(uint256 _number) public {
        secretNumber = _number;
    }

    function getSecretNumber() private view returns (uint256) {
        return secretNumber;
    }
}
```

### مثال 2: تعريف دالة خاصة
```solidity
pragma solidity ^0.8.0;

contract MyContract {
    uint256 private total;

    function add(uint256 _value) public {
        total += _value;
        logTotal();
    }

    function logTotal() private view {
        // دالة خاصة لتسجيل القيمة
    }
}
```

## الشرح
### المخاطر الشائعة
- **عدم إمكانية الوصول**: إذا كنت بحاجة إلى الوصول إلى المتغيرات أو الدوال من عقد آخر أو من عقد وراثي، فإن تعريفها كـ "private" سيكون عائقًا.
- **صعوبة في الاختبار**: يمكن أن تجعل الدوال المتعمدة كـ "private" من الصعب اختبارها بشكل مباشر، مما يتطلب استخدام دوال عامة أو أداة مساعدة.

### ملاحظات إضافية
- يمكن استخدام "internal" كبديل لـ "private" إذا كنت تريد السماح للعقود الوراثية بالوصول إلى المتغيرات أو الدوال.
- تأكد من فهم كيفية تأثير الوصول المحدود على تصميم العقد الذكي قبل استخدام "private".

## ملخص من جملة واحدة
تحدد خاصية "private" في Solidity مستوى الوصول للمتغيرات والدوال، مما يضمن أمان البيانات عبر تقليص نطاق الوصول إلى نفس العقد فقط.
