# الكلمة المفتاحية "virtual" في لغة سليديتي: شرح شامل

## ملخص
تُستخدم الكلمة المفتاحية "virtual" في لغة سليديتي لتحديد الدوال التي يمكن أن يتم تجاوزها (override) في العقود الوراثية. يتيح ذلك للمطورين إنشاء هياكل مرنة وقابلة لإعادة الاستخدام في العقود الذكية.

## الوثائق
### الغرض
تساعد الكلمة "virtual" في تحديد طريقة عمل الدوال في العقود الذكية. من خلال استخدام هذه الكلمة، يمكن للمطورين تحديد الدوال التي يمكن تجاوزها في العقود المشتقة، مما يزيد من القدرة على التخصيص والتطوير.

### الاستخدام
عند تعريف دالة في عقد، يمكن وضع الكلمة "virtual" قبل نوع الإرجاع للدالة. هذا يجعل الدالة قابلة للتجاوز في العقود التي ترث من العقد الأصلي.

#### صيغة الاستخدام:
```solidity
function functionName() public virtual returns (returnType) {
    // body of the function
}
```

### التفاصيل
- **تجاوز الدوال**: عند استخدام الكلمة "virtual"، يمكن لعقد آخر يرث العقد الأصلي تجاوز هذه الدالة باستخدام الكلمة "override". 
- **التوافق**: يجب أن تتطابق توقيعات الدوال في العقد المشتق مع الدالة الأصلية لتكون عملية التجاوز صحيحة.
- **الوصول**: يمكن أن تكون الدالة "virtual" عامة، داخلية، خاصة، أو محمية (public, internal, private, or protected).

## الأمثلة
### مثال بسيط
```solidity
pragma solidity ^0.8.0;

contract Base {
    function greet() public virtual returns (string memory) {
        return "Hello from Base";
    }
}

contract Derived is Base {
    function greet() public override returns (string memory) {
        return "Hello from Derived";
    }
}
```
في هذا المثال، يتم تجاوز الدالة `greet` في العقد `Derived` لتقديم رسالة مختلفة.

## الشرح
### الأخطاء الشائعة
- **عدم استخدام الكلمة "override"**: إذا تم تجاوز دالة "virtual" دون استخدام الكلمة "override"، سيظهر خطأ في الترجمة.
- **عدم المطابقة في التوقيع**: إذا كان هناك اختلاف في توقيع الدالة بين العقد الأصلي والعقد المشتق، سيؤدي ذلك أيضًا إلى أخطاء.

### ملاحظات إضافية
- يوصى باستخدام الكلمة "virtual" في الدوال التي من المحتمل أن تتطلب تغييرات في العقود المشتقة، مما يسهل في المستقبل توسيع العقود الذكية.
- يمكن استخدام الكلمة "virtual" مع الدوال، ولكن ليس مع المتغيرات.

## ملخص جملة واحدة
تُستخدم الكلمة المفتاحية "virtual" في سليديتي لتحديد الدوال التي يمكن تجاوزها في العقود المشتقة، مما يعزز قابلية التخصيص والتوسع في تطوير العقود الذكية.
