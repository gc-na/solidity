# Solidity에서의 생성자(Constructor) 이해하기

## 개요
Solidity에서 생성자는 스마트 계약의 초기화 시 호출되는 특별한 함수로, 계약의 상태 변수를 설정하거나 초기값을 지정하는 데 사용됩니다. 생성자는 계약이 배포될 때 자동으로 실행되며, 계약의 생애주기 동안 단 한 번만 호출됩니다.

## 문서화
### 목적
생성자의 주요 목적은 스마트 계약의 인스턴스가 생성될 때 필수 초기값을 설정하는 것입니다. 이를 통해 계약의 상태를 초기화하고, 배포된 후 불필요한 오류를 방지할 수 있습니다.

### 사용법
생성자는 `constructor` 키워드를 사용하여 정의됩니다. 생성자는 반환값이 없으며, 계약의 이름과 동일한 이름으로 정의됩니다. 생성자는 선택적으로 매개변수를 가질 수 있으며, 이를 통해 계약의 상태 변수에 값을 초기화할 수 있습니다.

```solidity
pragma solidity ^0.8.0;

contract MyContract {
    uint public value;

    // 생성자
    constructor(uint _value) {
        value = _value; // 상태 변수 초기화
    }
}
```

### 세부 사항
- **단 한 번 호출**: 생성자는 계약이 배포될 때 단 한 번만 호출되며, 이후에는 호출되지 않습니다.
- **가시성**: 생성자는 기본적으로 `public` 가시성을 가지며, 다른 가시성 수정자는 사용할 수 없습니다.
- **상태 변수 초기화**: 생성자 내에서 상태 변수를 초기화하고, 초기값을 설정할 수 있습니다.
- **상속**: 부모 계약의 생성자는 자식 계약에서 호출할 수 있으며, 이를 통해 계층적 계약 구조를 구현할 수 있습니다.

## 예제
```solidity
pragma solidity ^0.8.0;

contract ExampleContract {
    string public name;
    uint public age;

    // 생성자
    constructor(string memory _name, uint _age) {
        name = _name;
        age = _age;
    }
}

// 계약 배포 시: new ExampleContract("Alice", 30);
```

## 설명
- **공통적인 함정**: 생성자를 정의할 때, 계약의 상태 변수를 초기화하지 않으면 초기값이 `0` 또는 `false`로 설정됩니다. 이는 예상치 못한 결과를 초래할 수 있습니다. 
- **가변 인자**: 생성자에 변수를 전달할 때, 데이터 타입과 순서를 정확히 맞추는 것이 중요합니다. 잘못된 타입이나 순서로 인한 오류는 계약 배포 시 실패로 이어질 수 있습니다.
- **상속 시 주의**: 부모 계약에서 생성자가 매개변수를 요구하는 경우, 자식 계약에서 이를 호출해야 하므로 주의가 필요합니다.

## 한 줄 요약
Solidity의 생성자는 계약을 배포할 때 상태 변수를 초기화하는 데 사용되는 특별한 함수입니다.
